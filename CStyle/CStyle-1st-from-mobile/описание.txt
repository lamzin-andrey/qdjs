Зачем.
Чтобы писать  на js код, который потом легко портировать в код C. 

Файлы. 
c.js
Тут стандартые функции, которые можно легко и просто портировать с C to js. 
Например, strlen. Возможно, typedef (как переменная). 

stable.js - здесь мои функции, которые я уже реализовал и отладил на C. 
Могут внутри себя быть реализованы на нормальном js. 
Могут быть файлы с префиксом stable, stablephp.js например. 

Типы данных. 
Для указателей вместо звёздочки используем символ z. intz, charz, doublez, FILEz - это легко вернуть в настоящий C автозаменой. 
При объявлении переменной использовать вынос всех переменных в начало функции.
Тип пишем как отдельную переменную.
var char, ch = '\t';
var charz, s = "Hello", q = "World";
var charzz, arr = explode("o", s);

Функции. 
Типы аргументов в однострочном комментарии каждый на отдельной строке. 
Или все в одной линии, через запятую? 
Тип return идёт первым. 
Но транслировать легче такое:
/*tchar**t*/ function explode (/*tchar*t*/ del, /*tchar*t*/ str, /*tint*t*/ L, /*tRecord*t*/ rc);


Передача по ссылке. 
Используем комментарий, начинающийся с /*r и заканчивающийся на  r*/
Тогда он будет расскомментирован, а следующая строка удалена. 

/*r *L = qntReplace; r*/
L.v = qntReplace;
